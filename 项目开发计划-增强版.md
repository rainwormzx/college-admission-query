# 高考志愿填报查询系统实施计划（增强版）

## 项目概述
基于多源数据开发智能高考志愿填报查询系统，整合录取数据、院校详情、专业介绍、就业信息、学科评估等全方位数据，提供查询、详情展示、智能对比和志愿推荐功能。

---

## 数据源分析

### 1. 核心录取数据
- **文件**: `22-25年全国高校在浙江的专业录取分数.xlsx`
- **字段**: 年份、院校名称、专业、最低分数、最低位次、科类、批次等（17个字段）
- **用途**: 查询、筛选、推荐

### 2. 院校详情数据
- **文件**: `00、志愿填报必备资料\2、院校介绍\`
- **包含内容**:
  - 3000所高校基本信息（院校代码、所在地、类型、办学层次）
  - 保研率信息
  - 就业流向
  - 满意度评分
  - 部属院校名单
- **关键字段**: 院校名称、院校代码、所在地、类型、层次、保研率、满意度、就业率

### 3. 专业详情数据
- **文件**: `00、志愿填报必备资料\3、专业介绍\`
- **包含内容**:
  - 本科专业目录（完整版）
  - 专业-职业对照表
  - 专业就业信息（就业率、薪酬、职业方向、行业分布）
  - 专业满意度
  - 专业排名信息
- **关键字段**: 专业名称、专业代码、学科门类、就业率、平均薪酬、就业方向

### 4. 全国院校专业详情
- **文件**: `00、志愿填报必备资料\4、全国院校详情\`
- **包含内容**:
  - 2024年本科/专科/职高专业详情
  - 全国高等院校开设专业汇总
  - 2024中国大学数据
  - 2023-2024中国大学排名800强
  - QS/USNews世界大学排名

### 5. 学科评估数据
- **文件**: `00、志愿填报必备资料\5、各省第四轮学科评估名单\`
- **包含内容**: 31个省份的学科评估结果（PDF格式）
- **用途**: 专业强弱对比参考

### 6. 大学排名数据
- **文件**: `00、志愿填报必备资料\1、大学排名\`
- **包含内容**: 软科排名、QS排名、USNews排名、专业排名

## 技术选型

### 前端
- **框架**: React 18 + TypeScript
- **UI组件库**: Ant Design 5（企业级UI，适合数据表格和表单）
- **数据可视化**: ECharts（图表库）+ Apache ECharts for React
- **状态管理**: React Context + hooks（轻量级状态管理）
- **构建工具**: Vite
- **HTTP客户端**: axios

### 后端
- **框架**: Node.js + Express + TypeScript
- **数据库**: PostgreSQL（关系型数据库，适合复杂查询）
- **ORM**: Prisma（类型安全的ORM）
- **数据导入**: pandas (Python脚本) + node-xlsx

### 其他工具
- **Excel导出**: exceljs
- **API文档**: Swagger/OpenAPI
- **代码规范**: ESLint + Prettier

### 数据库表结构

#### 1. admission_data（录取数据表）
```sql
CREATE TABLE admission_data (
  id SERIAL PRIMARY KEY,
  year INTEGER NOT NULL,
  university_name VARCHAR(200),
  university_code VARCHAR(20),
  category VARCHAR(50),
  batch VARCHAR(50),
  subject_requirement VARCHAR(100),
  major VARCHAR(200),
  major_code VARCHAR(50),
  major_group VARCHAR(100),
  major_note TEXT,
  admission_count INTEGER,
  min_score INTEGER,
  min_rank INTEGER,
  school_location VARCHAR(100),
  school_nature VARCHAR(50),
  is_985 BOOLEAN,
  is_211 BOOLEAN
);
CREATE INDEX idx_year ON admission_data(year);
CREATE INDEX idx_major ON admission_data(major);
CREATE INDEX idx_min_score ON admission_data(min_score);
CREATE INDEX idx_min_rank ON admission_data(min_rank);
CREATE INDEX idx_university_name ON admission_data(university_name);
```

#### 2. university_details（院校详情表）
```sql
CREATE TABLE university_details (
  id SERIAL PRIMARY KEY,
  university_name VARCHAR(200) UNIQUE,
  university_code VARCHAR(20),
  province VARCHAR(50),
  city VARCHAR(50),
  school_type VARCHAR(50),              -- 院校类型（综合、理工、师范等）
  education_level VARCHAR(50),          -- 办学层次（本科、专科、高职）
  is_985 BOOLEAN,
  is_211 BOOLEAN,
  is_double_first_class BOOLEAN,       -- 是否双一流
  postgraduate_rate DECIMAL(5,2),      -- 保研率（%）
  employment_rate DECIMAL(5,2),        -- 就业率（%）
  satisfaction_score DECIMAL(3,1),     -- 满意度评分
  ranking_domestic INTEGER,            -- 国内排名
  ranking_qs INTEGER,                  -- QS排名
  ranking_usnews INTEGER,              -- USNews排名
  affiliation VARCHAR(100),            -- 隶属关系（教育部、省属等）
  website TEXT,                        -- 官网地址
  description TEXT,                    -- 学校简介
  employment_direction TEXT,           -- 就业方向
  founding_year INTEGER,               -- 建校年份
  campus_area INTEGER,                 -- 校园面积（亩）
  undergraduate_count INTEGER,         -- 本科生人数
  graduate_count INTEGER               -- 研究生人数
);
CREATE INDEX idx_university_details_name ON university_details(university_name);
CREATE INDEX idx_university_details_code ON university_details(university_code);
```

#### 3. major_details（专业详情表）
```sql
CREATE TABLE major_details (
  id SERIAL PRIMARY KEY,
  major_name VARCHAR(200) UNIQUE,
  major_code VARCHAR(20),
  category VARCHAR(50),                -- 学科门类
  sub_category VARCHAR(50),            -- 专业类别
  degree_type VARCHAR(50),             -- 授予学位
  duration INTEGER,                    -- 学制（年）
  employment_rate DECIMAL(5,2),        -- 就业率（%）
  avg_salary INTEGER,                  -- 平均薪酬（元）
  satisfaction_score DECIMAL(3,1),     -- 满意度评分
  ranking INTEGER,                     -- 专业排名
  core_courses TEXT,                   -- 核心课程
  employment_direction TEXT,           -- 就业方向
  employment_industry TEXT,            -- 就业行业分布
  employment_region TEXT,              -- 就业地区分布
  description TEXT,                    -- 专业介绍
  related_occupations TEXT             -- 相关职业
);
CREATE INDEX idx_major_details_name ON major_details(major_name);
CREATE INDEX idx_major_details_code ON major_details(major_code);
CREATE INDEX idx_major_details_category ON major_details(category);
```

#### 4. university_major_rel（院校专业关系表）
```sql
CREATE TABLE university_major_rel (
  id SERIAL PRIMARY KEY,
  university_name VARCHAR(200),
  major_name VARCHAR(200),
  is_key_major BOOLEAN,                -- 是否重点专业
  discipline_rating VARCHAR(10),       -- 学科评估等级（A+、A、B+等）
  admission_score_avg INTEGER,         -- 平均录取分
  faculty_strength TEXT,               -- 师资力量
  description TEXT,                    -- 专业在本校的特色
  FOREIGN KEY (university_name) REFERENCES university_details(university_name),
  FOREIGN KEY (major_name) REFERENCES major_details(major_name)
);
CREATE INDEX idx_um_rel_university ON university_major_rel(university_name);
CREATE INDEX idx_um_rel_major ON university_major_rel(major_name);
```

## 核心功能实现逻辑

### 1. 多条件组合查询
- 支持专业、地区、分数、位次的任意组合
- 分数和位次支持范围查询（如：500-600分，10000-50000位）
- 支持模糊搜索（专业名称）
- 支持精确筛选（地区、科类、批次等）
- 结果可排序、分页
- **新增**: 查询结果显示详情按钮，点击查看院校和专业详情

### 2. 数据导出
- 基于当前查询条件导出Excel
- 使用exceljs库生成格式化的Excel文件
- 支持自定义导出字段

### 3. 数据统计图表
- **分数分布直方图**: 展示各分数段的院校/专业数量
- **地区统计图**: 展示各地区院校数量分布（饼图/柱状图）
- **专业热度排行**: 展示报考人数最多的专业（柱状图）
- **年份趋势图**: 展示多年份平均分数变化（折线图）

### 4. 志愿推荐
- **冲刺**: 分数/位次高于考生5-10分或位次低1000-5000
- **稳妥**: 分数/位次与考生相近（±3分或±1000位次）
- **保底**: 分数/位次低于考生10-20分或位次高5000-10000
- 推荐算法基于历史数据统计

### 5. 多年份对比
- 选择院校和专业后，展示2022-2025年的录取分数和位次变化
- 支持多个专业同时对比
- 使用折线图或表格展示趋势

### 6. 院校专业详情展示（新增）
- **院校详情**: 展示学校基本信息、排名、保研率、就业率、满意度、建校历史、校园面积等
- **专业详情**: 展示专业介绍、学制、学位、就业方向、薪酬水平、核心课程、就业行业分布等
- **关联展示**: 显示某院校的某专业是否为重点专业、学科评估等级、师资力量等
- **快速预览**: 鼠标悬停在查询结果上时，显示关键信息卡片

### 7. 智能对比功能（新增）
- **院校对比**:
  - 支持2-4所院校同时对比
  - 对比维度：排名、保研率、就业率、满意度、办学层次、学科评估等
  - 雷达图展示多维度对比
  - 自动标注各维度最优院校
- **专业对比**:
  - 支持2-4个专业同时对比
  - 对比维度：就业率、薪酬、满意度、排名、学制等
  - 柱状图对比关键指标
  - 显示专业的就业方向差异
- **同专业不同院校对比**: 对比同一专业在不同院校的实力（学科评估等级、排名等）

## 数据导入策略

### 数据源优先级
1. **高优先级（必需）**:
   - `22-25年全国高校在浙江的专业录取分数.xlsx` - 核心录取数据
   - `02-全国院校基础信息-带保研率.xlsx` - 院校基本信息
   - `《院校"专业-职业"对照表（2024年版）.xlsx` - 专业详情
   - `2024年院校专业详情(本科).xlsx` - 本科专业详情

2. **中优先级（重要）**:
   - `2023-2024中国大学排名800强完整榜单.xlsx` - 排名数据
   - `院校满意度.xlsx` - 满意度数据
   - `专业就业信息.xlsx` - 就业信息
   - `专业介绍及薪酬表.xlsx` - 薪酬数据

3. **低优先级（增强）**:
   - QS/USNews世界大学排名
   - 各省学科评估PDF数据
   - 热门专业分析报告

### 数据导入顺序
1. 先导入录取数据（`admission_data`表）
2. 再导入院校详情（`university_details`表）
3. 然后导入专业详情（`major_details`表）
4. 最后导入关系数据（`university_major_rel`表）

### 数据清洗规则
1. **名称统一**: 处理院校名称的别名、简称问题
2. **缺失值处理**: 合理填充或标记为null
3. **数据类型转换**: 确保数值、布尔值正确转换
4. **去重**: 避免重复数据导入
5. **验证**: 导入后进行数据完整性检查

## 快速启动指南

### 1. 安装PostgreSQL数据库
```bash
# 创建数据库
createdb admission_query
```

### 2. 配置后端
```bash
cd backend
npm install
# 编辑.env文件配置数据库连接
DATABASE_URL="postgresql://postgres:password@localhost:5432/admission_query"
```

### 3. 初始化数据库
```bash
npm run prisma:generate
npm run prisma:migrate
```

### 4. 导入数据
```bash
cd scripts
pip install -r requirements.txt
# 按顺序导入数据
python import-excel.py           # 录取数据
python import-university.py      # 院校详情
python import-major.py           # 专业详情
python import-relation.py        # 关系数据
```

### 5. 启动后端
```bash
cd backend
npm run dev
```

### 6. 启动前端
```bash
cd frontend
npm install
npm run dev
```

### 7. 访问应用
浏览器打开：http://localhost:5173
